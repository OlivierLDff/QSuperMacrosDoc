Detailed Description

Macros to generate Auto Property uses either T or T* and is capable of adding constant-reference by deciding itself which type is the cheapest (using some template trickery internally).

Generate a Getter in the form Get<Name> To have a Qt-ish Getter (ie <name>()), define QSUPER_MACROS_USE_QT_GETTERS in your build system

Parameters
typeType of the attribute (int, quint32, QObject*, QString, etc...)
nameAttribute name in lowerCamelCase
NameAttribute name in UpperCamelCase

It generates for this goal :

// Default Naming Convention
CheapestType<type>::type_def GetName(void) const { return _name; }
// Qt Naming Convention
CheapestType<type>::type_def name(void) const { return m_name; }

You can declare a property in your QObject like this

// Create an integer with default to {}
QSM_CONSTANT_AUTO_PROPERTY(int, myInt, MyInt);
// Create a QString with default to {}
